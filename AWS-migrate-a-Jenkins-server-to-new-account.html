AWS migrate a Jenkins server to new account

I have configured a Jenkins server on an EC2 instance and wanted to migrate the Jenkins (with all settings of jobs, git configuration...). The first option I was thinking of is to create new EC2 instance on the other account, install Jenkins (with ChocoSetupBuildServerWithJenkins_Net45X.ps1) and copy everything (JENKINS_HOME folder). While doing the installation, Jenkins file couldn't be downloaded due to slowness of network (via Chocolaty). It's resolved by manually download the file and copy to cache folder C:\ProgramData\chocolatey\lib\jenkins\tools.


...but that didn't work as expect.

After a while of searching, I switched to other option: migrate the whole server EC2 instance. This was much faster. Below are steps:

1/ Create an image from existing EC2 instance.

Go to old account and create AMI image as private.

2/ Share the image with other account

Wait until the image is ready, select Modify Image Permissions to add target account number:


3/ Create new instance on other account

On target account console, go the the SAME region with old instance, create new EC2 instance from template in My AMIs & select Shared with me filter.

4/ New instance ready

The new instance is ready, but it will be required to update security group to make all connectivitys work. Then sign in on old account & delete the image, termiate the server...
