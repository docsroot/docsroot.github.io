A/ For evolutions

a1. How to gain disk space on servers

I have servers that log files / backup files are increasing, and sometime it could consume almost of spaces of the drive but this is a must to keep them. The servers are Windows with Powershell installed. So I have done:

  • Develop a script which identifies the files to be archived (compress them maximally and delete raw files)
  • Keep raw files (don't archive) within specific days for easiler accessing when needed

So the solution is: Powershell + 7-zip

B/ For the fixes and mitigrations

b1. How to fix log file of tempdb when it reaches to max size?

When this happens, it's not possible to use UI (through SSMS) to check it / edit the size. So below are scripts needed to handle:

Watch for current used % of log files

DBCC SQLPERF(logspace)


Get information about all files

select * from sys.master_files

or files of one database

select * from  dbo.sysfiles

After all check (if the drive have enough space to add onto a file) use the below command:

USE master;
GO
ALTER DATABASE DB01 
MODIFY FILE
    (NAME = DB01_log,
    SIZE = 300MB);
GO
